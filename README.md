# Test
команды для командной строки git bash

переход в нужную папку
cd <название папки>

создание репозитория
git clone https

запрос статуса гит
git status

настройка
git config - global user.name "<ваше имя>"
git config - global user.email "<адрес почты>"

если есть новый файл или изменения каких-либо файлов, его нужно добавить в коммит
точка означает «добавить все изменённые и неотслеживаемые файлы в подготовленные».
git add .

добавить инфо о внесенных изменениями
git commit -m 'сообщение о внесенном изменении'

обновить локальные изменения с веб версией
git push

добавить конкретный файл под версионный контроль
git add <имя файла>

контрольная сумма SHA-1 - это хеш-сумма изменений по алгоритму криптокрафического шифрования

удалить файл (написать имя файла с расширением)
git rm "имя файла"
git rm "dzen1.txt"
после этого git commit -m "удален файл", git push

команда, показывающая историю изменений. после этой команды для выхода нажать q / ctrl+x / ctrl+z
git log

создание новой ветки , имя ветки без кавычек
git branch testbrach00

 список веток, которые существуют в вашем локальном репозитории
git branch --list

информацию о последних коммитах каждой из веток
git branch --list -v

 для переключения на созданную ветку
git checkout testbranch00

файлы, находящиеся в репозитории
dir

файлы, находящиеся в репозитории (изменения на ветке main)
dir
